# Lab 08 - Azure Load Balancer

## Objective

The objective of this lab was to explore Azure Load Balancer and understand how Azure distributes incoming network traffic across multiple backend resources to improve availability, scalability and fault tolerance.

---

## Resources Used

- Azure Load Balancer (Standard SKU)
- Public IP Address
- Resource Group
- Azure Virtual Machine
- Azure Virtual Network

---

## Tasks Completed

- Navigated to Azure Load Balancers
- Created a new Standard Public Load Balancer
- Selected the existing Resource Group
- Configured the frontend IP configuration
- Created a dedicated Public IP for the Load Balancer
- Reviewed Backend Pools
- Reviewed Inbound Rules
- Reviewed Outbound Rules
- Reviewed Tags
- Validated configuration
- Successfully deployed the Load Balancer
- Reviewed the deployed Load Balancer
